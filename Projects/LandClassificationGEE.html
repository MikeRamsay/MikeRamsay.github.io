New Orleans Land Classification

Classifying satellite imagery using Google Earth Engine

March 2025

Project Summary

The goal of this project was to explore the land classification process and perform geospatial processing at scale with Google Earth Engine. To do this I used the internal Sentinel 2 satellite dataset in Google Earth Engine. Then, I ran a script to filter for satellite imagery with less than 10% cloud cover and set the date range to the 2024 calendar year. The next step was to build an image classifier which classifies the satellite image based on inputted training points. The process of adding these training points was relatively straightforward, consisting of manually adding points with the geometry import function to their proper classification on the satellite imagery. I added around 10 points for each feature class. The map to the left, on the bottom, only has 4 feature classes; urban, bare, water and vegetation. However, the top image has 2 additional feature classes, being two vegetation classifications, one for trees and one for grass, and the urban class was also split into roads/pavement and buildings. These additional classifications create an image with a greater level of detail as major roadways and open fields become distinguishable.


A real-world application for this type of mapping that comes to mind is flood risk analysis or permeable surface mapping. Something that is particularly important in cities like New Orleans which are at a greater risk to flood water-related events. This project also brought to light the impacts and responsibilities that geographers bare when conducting land classification analysis, especially when using training data and classes to speed up the classification process. Being aware of these implications is particularly important when this work has real-world impacts on the environment and people.


Classifying Healthy Vegetation using Normalized Difference Vegetation Index (NDVI)

For the next portion of this project, I classified healthy vegetation in the Sentinel-2 imagery by calculating the Normalized Difference Vegetation Index or NDVI. This is a measure of vegetation health and density which is calculated from the formula: NDVI = (NIR - Red) / (NIR + Red). The healthier vegetation contains more chlorophyll which reflects the most near-infrared (NIR) and green light. For Sentinel-2, spectral band 8 contains NIR wavelengths and Band 4 contains red wavelengths. So, I used the formula (Band 8 - Band 4)/(Band 8 + Band 4) in my script to calculate the NDVI. Google Earth Engine has a built in function (image.normalizedDifference) which calculates this function with the given values:


var ndvi = image.normalizedDifference(['B8', 'B4']).rename('ndvi');


This produced the NDVI layer for the given date range. To achieve the animated GIF, I captured NDVI layers for each month of 2024 to showcase the change in vegetation throughout the year. The graph below showcases these yearly changes in NDVI for this region. January-March appears to have the lowest average NDVI values, indicating less vegetation which is typical during the winter months. However, New Orleans is located by the coast and experiences consistent rainfall year round which most likely has an impact on the range of NDVI values in the graph or lack thereof.
